Función de coste¶

Se crea la funcin de coste, el objetivo ser ir minimizando el error hasta que llegue un punto en el que por ms iteraciones que llevemos a cabo, la minimizacin del error sea imperceptible. Sigue la siguiente frmula:

$(w,b)=1nn1(yiy^i)=1nn1((yilog(y^i)+(yi+1)log(1y^i))$

In [9]:
def compute_cost(theta, x, y): # creacion de la funcin de costes, est basada en el error, es decir, la diferencia entre el valor real y la prediccion
    m = len(y)
    y_pred = logistic_function(np.dot(x , theta))
    error = (y * np.log(y_pred)) + ((1 - y) * np.log(1 - y_pred))
    cost = -1 / m * sum(error)
    gradient = 1 / m * np.dot(x.transpose(), (y_pred - y))
    return cost[0] , gradient

Función de gradiente¶

Para encontrar el minimo, se utliza el descenso de gradiente, que se vale de derivadas para encontrar el mnimo. Ser esencial indicarle el alpha o tasa de aprendizaje y el nmero de iteraciones, en nuestro caso elegimos un alpha de 0.8 y 1500 iteraciones

In [11]:
def gradient_descent(x, y, theta, alpha, iterations): # Creamos la funcion de gradiente, en esta funcion incluimos entre otras cosas el alpha o learning rate que estara entre 0 y 1; y el nuemro de iteracciones o 'pasos'                                                    
    costs = []
    for i in range(iterations):
        cost, gradient = compute_cost(theta, x, y)
        theta -= (alpha * gradient)
        costs.append(cost)
    return theta, costs
theta, costs = gradient_descent(X, y, theta_init, 0.8, 1500) # aqui es donde le indicamos la tasa de aprendizaje y el numero de iteraciones

print("Theta despus del descenso de gradiente:", theta)
print("Resultado de coste:", costs[-1])
